    GoJobs!

    Many organisations (government or non-government) use websites to share information of new recruitment for the workers. This information overflows on thousands of sites with various attributes and criteria. However, this availability forms a complex puzzle in the selection process and leads to inefficient runtime. This project proposes a simple method for job searching simplification through a construction of web scraping technique on search engine. This study is resulting in an effective and efficient application for users to seek a potential job that aligns with their interests.

    Proposed Solution

    The proposed system is to make a website for searching multiple online job portals and providing job offers. This way the process of searching and finding jobs will be simplified. So instead of going to each of the job listing websites separately, our system will provide a platform to go through all job listings at one place, which will make it more convenient.

    Inventory Management
     The Inventory management system’s main goal is to simplify the task of a shop owner, mainly maintenance of stocked products, whether those products are company assets, raw materials and supplies, or finished products ready to be sold. An owner who has multiple numbers of shops can easily track the progress and sales of every shop without the need of physically going to the location and enquiring with the staff. The complete Inventory Management System includes the following features: A system for identifying every inventory item by using barcodes or unique identification codes. A billing software that is linked with this Inventory Management System (this project does not include the billing system). A simple interface that lets users connect and make changes with a central database. A special admin account which lets the user make logical changes in the database. Inform and provide the owner with timely reminders on which item requires restocking.

    Analyze the sales of a shop and provide a detailed report and list :

    • Most sold goods
    • Sales ratio of various shops
    • Monthly profits
    • Frequency of restocking of goods

    This following module shows one central database that consists of various branches of the shop where each shop maintains a separate list of goods being sold. The products database is simplified into categories to provide an easier user experience. A list of all employees working at the various locations is maintained in a database which is confidential only to the owner. Provides every employee with view-only access. The database updates simultaneously when billings are made.

    HealthYoda


    This project was a solution to one of the problem statements that was given at Smart India Hackathon. We qualified for the final round held at NIT trichy where the top 4 teams participated in the 36hour hackathon to come up with the most apt solution to the probelem
    Our solution was an android app that took an input for various medical fields like sugar , blood pressure etc the values were used to build a knn regressor model which predicted a value for calories that had to be burnt. Using a naive bayes model it predicted a suitable excercise for that particular value of calorie to be burnt.
    A php server was used to communicate to and from the app
    The android app was built using Native Android.
